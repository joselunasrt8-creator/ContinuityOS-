# Legitimacy IR Explainer

Status: Public Narrative Draft

## Core Idea

MindShift treats legitimacy like a compiler problem.

Human intent is not executed directly.

Intent is first converted into structured machine-readable objects.

## Simplified Flow

```text
intent
→ ATAO
→ authority binding
→ AEO
→ validation
→ execution boundary
→ proof
```

## What Is AEO?

AEO means:

```text
Action Execution Object
```

But structurally, AEO behaves like an intermediate representation (IR).

Just like compilers convert source code into structured intermediate forms before execution, MindShift converts intent into deterministic legitimacy objects.

## Why This Matters

Without structure:

- AI output is ambiguous
- authority is unclear
- replay protection breaks
- proof becomes inconsistent
- federation becomes unsafe
- execution becomes difficult to govern

MindShift solves this by forcing intent through deterministic legitimacy structure before execution exists.

## Core Rule

```text
Capability is not permission.
```

## Canonical Invariant

```text
If no valid object exists
→ nothing happens
```

## One Sentence Summary

MindShift turns intent into compiler-grade legitimacy objects before reality-changing execution is allowed.
